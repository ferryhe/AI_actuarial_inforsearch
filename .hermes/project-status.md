- Frontend: Chat document sidebar now supports selecting multiple markdown documents for comparison. The compare action switches to `comparison` mode and POSTs the selected markdown files through the backend's existing `document_sources` contract.
- Backend: Added all-mode KB syncing that includes every eligible markdown file with ready chunks for the selected chunk profile. Create-index requests and direct RAG indexing tasks now resync category/all-mode membership before choosing incremental files to embed.
- Backend review fix: Index-task responses now include `category_sync`, `all_sync`, and `chunk_bindings` only when those actions actually happened, preserving the manual-mode API shape.
- Backend: KB file membership changes now track `index_dirty_at`. Adding files marks the KB dirty so incremental indexing performs embedding/indexing for only newly pending files; removing files marks the KB dirty so the UI/API requires reindex before chat can trust the RAG index.
- Backend: Force reindex now clears persisted FAISS metadata and `rag_chunks` for the KB before rebuilding the current file set, and it can run with zero files so deleting all files clears stale RAG content.
- Backend: RAG index update semantics were tightened to avoid unnecessary full rebuilds. Removing a file from a KB now soft-deletes matching FAISS metadata vectors and removes that file's `rag_chunks` without forcing a rebuild. Re-indexing an updated file appends the new vectors and soft-deletes the old vectors for that file, implementing update as delete+insert. Incremental indexing is rejected when the existing index embedding provider/model/dimension differs from the configured runtime embedding.
- Backend: `VectorStore.remove_vectors()` now ignores invalid/negative indices, is idempotent, returns the number of newly soft-deleted vectors, and `VectorStore.search()` filters soft-deleted metadata while over-fetching enough candidates to return active neighbors.
- Backend review fix: `VectorStore.search()` now uses a cached soft-delete count with a bounded per-query over-fetch cap so many soft-deleted vectors cannot expand every FAISS search to the whole index.
- Backend review fix: `force_reindex=True` now rebuilds the complete current KB file set even if a caller passes a subset of `file_urls`, avoiding the prior mismatch where all chunks/status were cleared but only a subset was re-embedded.
- Settings/Chat embedding migration: Existing AI routing already allows embeddings to be configured for first-time setup, but treats provider/model changes as a migration. The Settings warning now says affected KB indexes must be rebuilt before Chat can use them, and coverage verifies the embedding route remains editable while changed embeddings mark existing KBs as `needs_reindex` and unusable in Chat until full reindex.
- Backend: Public chat routes now still load an authenticated session/API token when one is present, so registered users use the configured registered quota instead of anonymous quota. Admin chat requests now bypass daily chat quota accounting.
- Backend review fix: Public-permission fast paths now check for an auth token/session cookie before loading auth context, avoiding unnecessary DB opens for fully anonymous public requests.
- Frontend review fix: Knowledge/KB detail action error formatting now uses a shared `formatApiErrorDetail` helper from `client/src/lib/api.ts`; Chat document rows expose button semantics and keyboard activation while preserving the nested compare toggle.
