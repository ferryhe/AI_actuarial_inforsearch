## Key Components

### Backend
- **Flask Route**: `/file_preview` - Renders template
- **API Endpoint**: `/api/rag/files/preview` - Returns data
- **Permission**: `files.read` (public when auth disabled)

### Frontend
- **Template**: `file_preview.html` - Split-pane layout
- **JavaScript**: `file_preview.js` - Preview logic (10KB)
- **CSS**: `file_preview.css` - Styling (6.5KB)

### Libraries
- **PDF.js 3.11.174**: PDF rendering
- **Marked.js 11.1.1**: Markdown parsing
- Both with SRI integrity checks

## User Flow

1. User clicks "Preview" button (RAG detail or database page)
2. Opens new tab with `/file_preview?file_url=...&kb_id=...`
3. Page loads template with loading spinners
4. JavaScript fetches data from API endpoint
5. API queries database (files, catalog_items, rag_chunks)
6. Returns JSON with file info, markdown, chunks
7. JavaScript renders:
   - Original file (PDF/image/download)
   - Markdown content
   - Chunks with metadata
8. User can:
   - Navigate PDF pages
   - Scroll through markdown
   - View chunk details
   - Download original file
