# FastAPI 文件读取接口第二批测试指南

## 测试目标

确认以下接口已由 FastAPI 原生提供，且 React 页面调用正常：

- `GET /api/sources`
- `GET /api/files/detail`
- `GET /api/files/{file_url}/markdown`

## 启动方式

### 1. 启动后端

```bash
.\.venv\Scripts\python.exe -m ai_actuarial api --host 127.0.0.1 --port 8000
```

预期：

- FastAPI 网关启动成功
- 未迁移接口仍可通过 legacy Flask fallback 使用

### 2. 启动前端

```bash
npm run dev
```

预期：

- Vite 开发服务器启动成功
- 浏览器可访问 `http://localhost:5173`

## 手工测试清单

### 1. Database 页面

- [ ] 打开 `http://localhost:5173/database`
- [ ] 确认 source 下拉框正常加载
- [ ] 确认 categories 下拉框正常加载
- [ ] 确认文件列表正常显示
- [ ] 修改 source/category/query 筛选后，列表仍能正常刷新

预期：

- `sources` 过滤项可见
- 页面无 401/500 弹窗
- 列表筛选逻辑与改动前一致

### 2. File Detail 页面

- [ ] 从 Database 点击任意文件进入详情页
- [ ] 确认标题、来源、分类、摘要等详情正常显示
- [ ] 确认 markdown 区域能正常显示内容或空状态
- [ ] 对一个存在 markdown 的文件，确认 markdown 内容加载成功

预期：

- `/api/files/detail` 返回的详情字段完整
- `/api/files/{file_url}/markdown` 返回结构正确
- 页面无空白或卡死

### 3. 带认证场景

如果当前环境启用了 `REQUIRE_AUTH=true`：

- [ ] 登录后打开 Database 页面
- [ ] 登录后打开 File Detail 页面
- [ ] 确认 source、详情、markdown 均可正常加载
- [ ] 注销或无 token 状态下重试直连 `/api/sources`

预期：

- 已登录状态可正常访问
- 未登录直连受保护接口时返回 `401`

### 4. migration status

- [ ] 打开 `http://127.0.0.1:8000/api/migration/status`
- [ ] 检查 `native_paths`

预期：

- `native_paths` 包含：
  - `/api/sources`
  - `/api/files/detail`
  - `/api/files/{file_url:path}/markdown`

## 如果发现问题

请优先记录：

1. 失败页面或失败接口
2. 浏览器 Network 面板里的请求 URL 和状态码
3. 返回体内容
4. 是否开启了 `REQUIRE_AUTH=true`
5. 使用的文件 URL 是否包含空格或 `%20`
